# Datenbank-Setup für Bildergalerie

## Erforderliche Tabellen und Konfiguration

### 1. Storage Bucket erstellen

Gehe zu Supabase Dashboard → Storage und erstelle einen neuen Bucket:
- **Bucket Name:** `user_gallery`
- **Public Access:** Ja (um URLs zu generieren)

### 2. Tabelle `user_gallery` erstellen

Führe folgende SQL in Supabase aus:

```sql
CREATE TABLE user_gallery (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  image_filename TEXT NOT NULL,
  caption TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(user_id, image_filename)
);

-- RLS Policies
ALTER TABLE user_gallery ENABLE ROW LEVEL SECURITY;

-- Alle können ihre eigenen Bilder lesen
CREATE POLICY "Users can read own gallery"
  ON user_gallery
  FOR SELECT
  USING (auth.uid() = user_id);

-- Alle können ihre eigenen Bilder hochladen
CREATE POLICY "Users can insert own gallery"
  ON user_gallery
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- Alle können ihre eigenen Bilder löschen
CREATE POLICY "Users can delete own gallery"
  ON user_gallery
  FOR DELETE
  USING (auth.uid() = user_id);
```

### 3. Tabelle `gallery_comments` erstellen

Führe folgende SQL in Supabase aus:

```sql
CREATE TABLE gallery_comments (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  gallery_id BIGINT NOT NULL REFERENCES user_gallery(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  comment_text TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- RLS Policies
ALTER TABLE gallery_comments ENABLE ROW LEVEL SECURITY;

-- Alle können Kommentare zu den Galerien lesen
CREATE POLICY "Anyone can read comments"
  ON gallery_comments
  FOR SELECT
  USING (true);

-- Alle können Kommentare hinzufügen
CREATE POLICY "Users can insert comments"
  ON gallery_comments
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- Alle können ihre eigenen Kommentare löschen
CREATE POLICY "Users can delete own comments"
  ON gallery_comments
  FOR DELETE
  USING (auth.uid() = user_id);
```

## Weiterleitung für Storage

Um die Bilder direkt zu servieren, stellsicher, dass die RLS-Policies für den Storage richtig konfiguriert sind:

Gehe zu Storage → Policies und füge folgende Policies für `user_gallery` Bucket hinzu:

```
- Users can upload to user_gallery → (*.jpg; *.jpeg; *.png; *.gif; *.webp) → With any RLS policies
- Users can read user_gallery → true
```

## Ready!

Nach dem Setup kan die Galerie-Funktion genutzt werden:
1. Nutzer können sich einloggen
2. Im "Mein Bereich" können Schüler Bilder hochladen
3. Bilder können kommentiert werden
